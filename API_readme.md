# Техническая документация для мини-приложения мониторинга транспорта Лесозаводска

## 1. Общие сведения и контекст

Мини-приложение разрабатывается для мессенджера **МАКС** для МБУ «ХОЗУ АЛМО» (г. Лесозаводск, Приморский край).

Приложение отображает движение общественного транспорта на карте OpenStreetMap с использованием библиотеки **Leaflet** и интегрируется с мессенджером через **MAX Bridge**.

Данные о маршрутах, направлениях, геометрии и транспортных средствах (ТС) предоставляются серверной частью АСУД через Public API.

---

## 2. Подключаемые внешние ресурсы

### Библиотека Leaflet (для работы с картой)
- **CSS:** `https://unpkg.com/leaflet@1.9.4/dist/leaflet.css`
- **JS:** `https://unpkg.com/leaflet@1.9.4/dist/leaflet.js`

### MAX Bridge (для интеграции с мессенджером МАКС)
- **JS:** `https://st.max.ru/js/max-web-app.js`

### Тайлы карты OpenStreetMap
- **URL-шаблон:** `https://{s}.tile.openstreetmap.org/{z}/{x}/{y}.png`
- Где `{s}` — поддомен (a, b, c), `{z}` — уровень зума, `{x}` и `{y}` — координаты тайла.

## 3. Public API АСУД

**Базовый URL:** `https://диспетчер-автобусов.рф/api/public`

Все эндпоинты поддерживают только методы `GET`, `HEAD`, `OPTIONS`. Авторизация не требуется.

### 3.1. Маршруты

**Эндпоинт:** `GET /routes`

Возвращает список всех маршрутов.

| Поле | Тип | Описание |
|---|---|---|
| `id` | string (UUID) | Уникальный идентификатор записи маршрута |
| `route_id` | string (UUID) | Идентификатор маршрута (совпадает с `id`) |
| `code` | string | Номер маршрута (например, `"111"`, `"10-5"`) |
| `name` | string | Название маршрута (например, `"Филаретовка - ДВГТУ"`) |

### 3.2. Направления маршрутов

**Эндпоинт:** `GET /routes/:routeId/directions`

Возвращает список направлений для конкретного маршрута.

| Поле | Тип | Описание |
|---|---|---|
| `id` | string (UUID) | Уникальный идентификатор записи направления |
| `direction_id` | string (UUID) | Идентификатор направления (совпадает с `id`) |
| `route_id` | string (UUID) | Идентификатор маршрута, к которому относится направление |
| `code` | string | Код направления (`OUTBOUND` или `INBOUND`) |
| `name` | string | Название направления (например, `"Филаретовка - ДВГТУ"`) |

### 3.3. Геометрия маршрутов (Шейпы)

**Эндпоинт:** `GET /routes/:routeId/shape?format=geojson`

Возвращает геометрию маршрута в формате GeoJSON для каждого направления.

| Поле | Тип | Описание |
|---|---|---|
| `direction_id` | string (UUID) | Идентификатор направления |
| `route_id` | string (UUID) | Идентификатор маршрута |
| `direction_code` | string | Код направления (`OUTBOUND` / `INBOUND`) |
| `direction_name` | string | Название направления |
| `length_m` | number / null | Длина маршрута в метрах |
| `updated_at` | string (ISO8601) | Дата последнего обновления геометрии |
| `geojson` | object | Объект GeoJSON |
| `geojson.type` | string | Тип геометрии (`LineString`) |
| `geojson.coordinates` | array | Массив координат. **Важно:** координаты идут в порядке `[долгота, широта]` |

### 3.4. Транспортные средства на карте

**Эндпоинт:** `GET /vehicles`

Возвращает список всех ТС с их текущим положением и статусами.

**Параметры запроса (опционально):**
- `routeId` — фильтр по ID маршрута
- `directionId` — фильтр по ID направления
- `bbox` — фильтр по bounding box в формате `minLon,minLat,maxLon,maxLat`

| Поле | Тип | Описание |
|---|---|---|
| `vehicle_id` | string (UUID) | Уникальный идентификатор ТС |
| `fleet_number` | string | Бортовой или государственный номер ТС |
| `lat` | number | Широта |
| `lon` | number | Долгота |
| `speed_kmh` | number | Скорость в км/ч |
| `course_deg` | number | Курс/направление движения в градусах (0–360, где 0 — север) |
| `position_source` | string | Источник позиционирования (`GLONASS` / `GPS`) |
| `status` | string | Статус ТС |
| `last_updated_at` | string (ISO8601) | Время последнего обновления данных на сервере |
| `last_fix_time` | string (ISO8601) | Время последнего фиксирования координат устройством |
| `late_sec` | number / null | Опоздание в секундах |
| `direction_id` | string (UUID) / null | ID текущего направления |
| `direction_code` | string / null | Код текущего направления |
| `route_id` | string (UUID) / null | ID текущего маршрута |
| `route_code` | string / null | Номер текущего маршрута |
| `route_name` | string / null | Название текущего маршрута |
| `trip_instance_id` | string (UUID) / null | ID текущего рейса |

### 3.5. Остановки и отправления

- **Поиск остановок:** `GET /stops/search?query=...`
- **Ближайшие остановки:** `GET /stops/near?lat=..&lon=..&radius=800`
- **Посадочные платформы:** `GET /stops/:placeId/platforms`
- **Отправления с платформы:** `GET /platforms/:platformId/departures?at=ISO&limit=30`
- **Отправления с остановки:** `GET /stops/:placeId/departures?at=ISO&limit=30` (дополнительно отдаются `predicted_departure_dt` и `delay_sec`).

## 4. MAX Bridge API

После подключения скрипта MAX Bridge становится доступен глобальный объект `window.WebApp`.

### 4.1. Инициализация и базовые данные

- `ready()` — сообщает мессенджеру о готовности интерфейса.
- `expandViewport()` — разворачивает приложение на весь экран.
- `initData` — строка со стартовыми параметрами (для валидации на бэкенде).
- `initDataUnsafe` — объект с распарсенными данными.
- `initDataUnsafe.start_param` — параметры, переданные через диплинк.
- `platform` — платформа запуска (`ios`, `android`, `desktop`, `web`).
- `version` — версия приложения МАКС.
- `colorScheme` — текущая тема интерфейса (`light` или `dark`).

### 4.2. Управление интерфейсом и устройством

- `requestScreenMaxBrightness()` / `restoreScreenBrightness()` — управление яркостью экрана.
- `enableClosingConfirmation()` / `disableClosingConfirmation()` — предупреждение при закрытии.
- `openLink(url)` — открытие ссылки во внешнем браузере.
- `openMaxLink(url)` — открытие диплинка внутри МАКС.
- `downloadFile(url, file_name)` — скачивание файла.
- `shareContent(params)` / `shareMaxContent(params)` — вызов экрана шеринга.
- `openCodeReader(fileSelect)` — открытие камеры для сканирования QR-кода.
- `requestContact()` — запрос номера телефона пользователя.

### 4.3. Кнопка «Назад» (BackButton)

- `show()` / `hide()` — управление видимостью.
- `isVisible` — статус отображения.
- `onClick(callback)` / `offClick(callback)` — обработчики нажатия.

### 4.4. Хранилища и модули

- `DeviceStorage` — локальное хранилище (`setItem`, `getItem`, `removeItem`, `clear`).
- `SecureStorage` — зашифрованное хранилище.
- `BiometricManager` — биометрическая аутентификация.
- `HapticFeedback` — тактильные отклики.
- `NfcManager` — работа с NFC (только Android).

## 5. Правила обработки данных и бизнес-логика

### 5.1. Фильтрация служебного транспорта

Транспортные средства, у которых поле `route_code` равно `null` или пустой строке, являются служебными автомобилями или ТС в резерве. Их **строго запрещено** отображать на публичной карте во избежание раскрытия местоположения служебного транспорта.

### 5.2. Системы координат

- API АСУД в формате GeoJSON возвращает координаты в порядке **[долгота, широта]** (стандарт GeoJSON).
- Библиотека Leaflet требует координаты в порядке **[широта, долгота]**.
- При отрисовке шейпов маршрутов и маркеров необходимо явно менять местами значения координат.

### 5.3. Ориентация иконок транспорта

Поле `course_deg` в данных ТС содержит угол направления движения в градусах (от 0 до 360, где 0 — это север). Иконки транспорта на карте должны поворачиваться ровно на этот угол.

### 5.4. Периодичность обновления

Данные о местоположении транспортных средств (`/vehicles`) должны запрашиваться и обновляться на карте каждые **5 секунд** для обеспечения плавности движения и актуальности информации.

### 5.5. Географический центр

Дефолтный центр карты при загрузке должен быть установлен в координаты центра города Лесозаводск:
- Широта: **45.4556**
- Долгота: **133.4056**

### 5.6. Требования к серверной части (CORS)

Поскольку мини-приложение размещается на стороннем хостинге (GitHub Pages), сервер АСУД должен отдавать заголовок:

```
Access-Control-Allow-Origin: *
```

(или явно разрешать домен хостинга) для эндпоинтов префикса `/api/public`, иначе браузер заблокирует запросы.

### 5.7. Адаптация под тему МАКС

Приложение должно проверять `window.WebApp.colorScheme` при запуске и, при необходимости, переключать стиль тайлов карты (например, на темную тему **CARTO**) и цвета UI-элементов для соответствия системной теме мессенджера.
